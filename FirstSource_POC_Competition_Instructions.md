## Selected POC: Incoming Request Processing Workflow

### Objective

Design and develop an AI-powered prototype that automatically receives, classifies, and processes incoming requests such as customer queries, complaints, or service requests.

The key requirement is multi-step branching logic: each request type should trigger a distinct remediation workflow instead of a single generic response.

For stronger alignment with TeleMedik / Firstsource, use a healthcare contact center framing where possible. Suitable request types include member complaints, appointment support, billing disputes, provider/network questions, benefits inquiries, medication or care coordination questions, and urgent escalations.

### Core Requirements

The prototype should:

- Accept an incoming request through a form, file upload, simulated inbox, or similar input.
- Use AI to classify the request by type and urgency.
- Branch into a type-specific remediation workflow.
- Execute at least two downstream steps per branch.
- Handle at least three distinct request types.
- Produce clearly legible outputs for an operations team.

### Minimum Branching Expectation

The solution should support at least three request types, each with its own remediation strategy.

Examples from the brief include:

| Request Type | Urgency | Example Remediation Steps | Expected Output |
| --- | --- | --- | --- |
| Complaint | High | Acknowledge receipt, escalate to senior handler, log priority case, set follow-up reminder. | Escalation notification, draft acknowledgement, case log entry. |
| General Enquiry | Low | Classify sub-topic, generate response from knowledge base, send response, log resolved status. | Auto-generated response and resolved status log. |
| Service Request | Medium | Extract required details, route to department, generate confirmation, set SLA timer. | Routing notification, confirmation message, SLA flag. |
| Escalation / Urgent | Critical | Flag for human review, draft urgent acknowledgement, notify supervisor, pause auto-resolution. | Supervisor alert, draft acknowledgement, human-in-the-loop flag. |

### Expected Output Per Request

Each processed request should produce:

- Classification label.
- Urgency level.
- Branch-specific action summary.
- Generated outputs such as:
  - Draft response.
  - Routing notification.
  - Follow-up task.
  - Escalation notice.
  - Case log entry.

### Optional Enhancements

The brief suggests optional enhancements such as:

- Batch processing multiple requests.
- Processing log or audit trail showing classification decisions and actions taken.
- Dashboard showing request volumes by type and status.
- Escalation override mechanism for uncertain AI classifications.

For over-delivery, prioritize enhancements that reinforce the company's likely pain points:

- Audit trail for every classification and action.
- Confidence score and human-review routing.
- SLA timers and follow-up reminders.
- Dashboard with request volume, urgency mix, branch outcomes, and unresolved escalations.
- Bilingual-ready architecture or sample Spanish/English request handling, if feasible.
- Healthcare-safe wording that avoids making clinical decisions and routes sensitive issues to human review.
